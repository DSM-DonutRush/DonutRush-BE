# Donut Rush — spec.md

## 1. 제품 개요

SDGs(정의·평화·효과적인 제도) 메시지를 담은 캐주얼 도넛 매칭 게임.
브럼프·이정은·부딘 중 캐릭터를 선택하고, 화면에 등장하는 도넛에 매핑된 방향키를 눌러 포만감 게이지를 90점까지 채우면 승리.
제한 시간은 모든 난이도 동일 2분. 싱글 플레이와 2~4인 실시간 멀티플레이 지원.

---

## 2. 사용자 플로우

```
[메인 화면]
    └─ [캐릭터 선택]
         └─ [난이도 선택] → Easy / Normal / Hard
              ├─ [싱글 플레이] → [게임 화면] → [결과 화면]
              └─ [멀티 매칭 대기] → [게임 화면] → [결과 화면]

[랭킹 페이지]   ← 상단 네비게이션
[마이페이지]    ← 상단 네비게이션
```

---

## 3. 도넛 시스템

### 도넛 속성
도넛은 3가지 속성의 조합으로 구성된다.

- **색상**: 핑크, 초코, 바닐라, 딸기, 블루베리 등
- **모양**: 링, 트위스트, 필드 등
- **토핑**: 스프링클, 초코칩, 글레이즈, 무토핑 등

### 도넛-키 매핑
- 게임 시작 시 도넛 4종을 랜덤 선택하여 방향키(↑↓←→)에 1:1 매핑
- 매핑은 게임 내내 고정

```
예시:
↑ = 핑크 스프링클 도넛
↓ = 초코 링 도넛
← = 바닐라 글레이즈 도넛
→ = 딸기 트위스트 도넛
```

---

## 4. 난이도 명세

### Easy
- 도넛 등장 시 **눌러야 하는 방향키 힌트 함께 표시**
- 매핑 정보가 화면에 항상 노출

### Normal
- **도넛만 표시**, 방향키 힌트 없음
- 플레이어가 매핑을 외워서 입력

### Hard — 시퀀스 모드
| 페이즈 | 설명 |
|--------|------|
| 제시 페이즈 | 도넛 4~7개(매 시퀀스마다 랜덤)가 순서대로 표시됨. 키 입력 불가 |
| 블라인드 전환 | 아무 방향키를 누르면 도넛이 모두 사라지고 검은 도넛 하나만 표시 |
| 입력 페이즈 | 기억한 순서대로 방향키 입력 |
| 채점 | 전체 시퀀스 순서 정답 시 점수 획득 |

---

## 5. 점수 및 패널티

| 상황 | 결과 |
|------|------|
| 정답 입력 | 점수 +1 |
| 오답 입력 | 2초간 키보드 입력 불가 (포만감 게이지 변화 없음) |

- 제한 시간: **모든 난이도 2분**
- 시간 종료 또는 90점 달성 시 게임 종료

---

## 6. 포만감 게이지 단계

| 점수 | 단계 | 캐릭터 외형 변화 |
|------|------|----------------|
| 10점 | 1단계 | 배가 약간 나옴 |
| 30점 | 2단계 | 좀 더 나옴 |
| 50점 | 3단계 | 많이 나옴 |
| 90점 | **클리어** | Full 상태 — 게임 종료 |

---

## 7. 멀티플레이

- 2~4인 랜덤 매칭
- 입장 시 난이도 선택 → **모든 플레이어 동일 난이도**로 진행
- WebSocket(STOMP)으로 각 플레이어의 포만감 게이지 실시간 동기화
- 승리 조건: 가장 먼저 90점(Full) 도달한 플레이어
- 결과 화면: 순위(1~4등) + 각자 클리어 시간 비교

---

## 8. 랭킹 시스템

| 타입 | 설명 |
|------|------|
| 전체 랭킹 | 전체 기간, 클리어 시간 오름차순 |
| 주간 랭킹 | 이번 주 기록만 집계 |

**표시 데이터**: 순위, 닉네임, 클리어 시간, 사용 캐릭터, 획득 뱃지

**정렬 기준**
1순위: 클리어 시간 오름차순
2순위: 최신 기록

**갱신**: 게임 종료 시 서버 자동 저장, 조회 시 최신 반환

---

## 9. 마이페이지

- 프로필: 닉네임, 프로필 이미지
- 플레이 기록: 총 플레이 횟수, 최고 기록, 평균 클리어 시간
- 뱃지 컬렉션: 획득/미획득 뱃지 목록
- 설정: 닉네임 변경, 로그아웃

---

## 10. 뱃지 시스템

| 뱃지 | 조건 |
|------|------|
| 기본 클리어 | 게임 완료 (성공) |
| 무실수 | 패널티 0회 클리어 |
| 스피드 | 일정 시간 이내 클리어 (기준 시간 TBD) |

## 11. API 명세

### POST /game/result — 게임 결과 저장
```json
요청:
{
  "userId": "string",
  "character": "브럼프 | 이정은 | 부딘",
  "clearTime": 12345,
  "success": true,
  "penaltyCount": 2,
  "difficulty": "EASY | NORMAL | HARD",
  "createdAt": "2025-06-01T12:00:00Z"
}

응답:
{
  "success": true,
  "message": "결과 저장 완료",
  "data": {
    "badgesEarned": ["기본 클리어", "스피드"]
  }
}
```

### GET /ranking?type=overall|weekly — 랭킹 조회
```json
응답:
{
  "success": true,
  "message": "",
  "data": [
    {
      "rank": 1,
      "nickname": "도휘",
      "clearTime": 8231,
      "character": "브럼프",
      "badges": ["무실수", "스피드"]
    }
  ]
}
```

### GET /user — 유저 정보 조회
### PUT /user — 닉네임 수정

## 12. 데이터 모델

### GameResult
```
id            BIGINT PK
userId        VARCHAR
character     ENUM('브럼프', '이정은', '부딘')
clearTime     BIGINT (밀리초)
success       BOOLEAN
penaltyCount  INT
difficulty    ENUM('EASY', 'NORMAL', 'HARD')
createdAt     DATETIME
```

### Badge
```
id          BIGINT PK
userId      VARCHAR
badgeType   ENUM('기본 클리어', '무실수', '스피드')
earnedAt    DATETIME
```

### User
```
id            VARCHAR PK
nickname      VARCHAR
profileImage  VARCHAR (nullable)
createdAt     DATETIME
```
